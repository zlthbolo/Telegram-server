## 📦 التثبيت المحلي

### 1. استنساخ المشروع
```bash
git clone https://github.com/yourusername/telegram-radar.git
cd telegram-radar
```

### 2. إنشاء بيئة افتراضية
```bash
python3.11 -m venv venv
source venv/bin/activate  # على Linux/Mac
# أو
venv\Scripts\activate  # على Windows
```

### 3. تثبيت المكتبات
```bash
pip install -r requirements.txt
```

### 4. إعداد متغيرات البيئة
```bash
cp .env.example .env
# ثم عدّل الملف وأضف قيمك الخاصة
```

### 5. تشغيل التطبيق
```bash
python app.py
```

التطبيق سيكون متاحاً على: `http://localhost:5000`

## 📱 إضافة حسابات تليجرام

### الخطوة 1: الحصول على بيانات API
1. اذهب إلى [my.telegram.org](https://my.telegram.org)
2. سجّل دخولك برقم هاتفك
3. انقر على "API Development Tools"
4. أنشئ تطبيقاً جديداً واحصل على:
   - **API ID** (رقم)
   - **API Hash** (نص طويل)

### الخطوة 2: إضافة الحساب في الواجهة
1. ادخل إلى لوحة التحكم
2. اذهب إلى قسم "إدارة الحسابات"
3. أدخل البيانات:
   - رقم الهاتف (مع مفتاح الدولة، مثل: 967XXXXXXXXX)
   - API ID
   - API Hash
   - مجموعة الإشعارات (اختياري)
4. انقر "إضافة حساب"

### الخطوة 3: التحقق من الهاتف
- النظام سيطلب رمز التحقق الذي يرسله تليجرام لهاتفك
- أدخل الرمز في الواجهة
- إذا كان الحساب محمياً بـ 2FA، أدخل كلمة المرور

## 🤖 الذكاء الاصطناعي (OpenRouter)

### التفعيل
1. احصل على مفتاح API من [OpenRouter](https://openrouter.ai)
2. اذهب إلى "إعدادات الذكاء الاصطناعي"
3. أدخل المفتاح وفعّل التصنيف الذكي
4. انقر "حفظ الإعدادات"

### كيفية التصنيف
- **Seeker (طالب)**: يطلب مساعدة أكاديمية
- **Marketer (معلن)**: يروج لخدمات تجارية

النظام يرسل الرسائل المصنفة كـ "seeker" فقط للمجموعة المستهدفة.

## 🚀 النشر على السحابة

### النشر على Render (الأسهل)

#### الخطوة 1: رفع الكود على GitHub
```bash
git add .
git commit -m "Initial commit"
git push origin main
```

#### الخطوة 2: إنشاء Web Service على Render
1. اذهب إلى [render.com](https://render.com)
2. انقر "New +" ثم "Web Service"
3. اختر مستودعك من GitHub
4. أدخل الإعدادات:
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `gunicorn app:app`

#### الخطوة 3: إضافة متغيرات البيئة
في لوحة Render، أضف:
```
SECRET_KEY=your-secret-key-here
ADMIN_EMAIL=admin@radar.com
ADMIN_PASSWORD=your-password
OPENROUTER_API_KEY=your-api-key
PORT=5000
```

#### الخطوة 4: النشر
انقر "Deploy" وانتظر انتهاء العملية.

### الحفاظ على التطبيق نشطاً 24/7

الخطة المجانية على Render تنام بعد 15 دقيقة من عدم النشاط. لحل هذا:

1. استخدم [UptimeRobot](https://uptimerobot.com)
2. أنشئ monitor جديد:
   - **Type:** HTTP(s)
   - **URL:** رابط تطبيقك على Render
   - **Interval:** 5 دقائق
3. هذا سيبقي التطبيق مستيقظاً طوال الوقت

## 🔧 استكشاف الأخطاء الشائعة

### المشكلة: "Invalid API credentials"
**الحل:** تأكد من صحة API ID و API Hash من my.telegram.org

### المشكلة: "Session password needed"
**الحل:** الحساب محمي بـ 2FA. أدخل كلمة المرور عند الطلب.

### المشكلة: "Flood wait error"
**الحل:** النظام يرسل رسائل كثيرة. انتظر قليلاً قبل إرسال المزيد.

### المشكلة: "OpenRouter API error"
**الحل:** تحقق من صحة مفتاح API وأن لديك حصة مجانية متبقية.

### المشكلة: الرسائل لا تُرسل للمجموعة
**الحل:** 
- تأكد من تفعيل الرادار
- تأكد من وجود كلمات مفتاحية
- تحقق من أن مجموعة الإشعارات محددة بشكل صحيح
